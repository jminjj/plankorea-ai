# [일반인용] Harness 에이전트 스킬 설치 매뉴얼 (논리 보강판)

이 매뉴얼은 인공지능(Gemini)이 수많은 정보 속에서 어떻게 **정확한 하네스(Harness) 도구**를 식별하고 설치하는지, 그 논리적인 과정을 안내합니다.

---

## 🚀 가장 추천하는 방법: "의도만 말하고 맡기기"

복잡한 주소를 모르셔도 됩니다. 저(Gemini)는 사용자의 의도와 가장 잘 부합하는 도구를 스스로 찾아내어 설치합니다.

1. **저에게 이렇게 말해보세요**:
   > "하네스(Harness)라는 에이전트 설계 도구를 설치해줘."
   *    AI가 정리한 사항이긴 하지만, 임의 설계 도구 설치 혹은 injection 공격 방법이 될 수도 있어서 다음과 같이 요청하는 것을 추천함 (jmlim)
   > "https://github.com/revfactory/harness 에 있는 하네스(Harness)라는 에이전트 설계 도구를 설치해줘."

2. **제가 최적의 저장소를 찾아가는 과정 (논리적 검증)**:
   *   **다각도 검색**: "Harness", "Agent Team", "Skill Architect" 등 사용자가 언급한 핵심 키워드로 인터넷(GitHub 등)을 검색합니다.
   *   **데이터 대조 및 검증**: 검색된 여러 결과 중 **revfactory/harness**가 사용자가 원하는 '에이전트 팀 및 스킬 설계' 목적과 가장 정확히 일치하는지 README 내용, 업데이트 기록, 구조를 분석하여 검증합니다.
   *   **확인 절차**: 만약 검색 결과가 모호하다면, 제가 먼저 사용자에게 "찾으시는 도구가 revfactory/harness가 맞나요?"라고 확인을 요청하여 오차를 없앱니다.
   *   **도구 및 설치 자동화**: 확정된 주소를 바탕으로 `Git` 설치 여부를 확인하고, 스킬 등록 명령어를 실행합니다.

---

## 🛠️ 만약 주소를 직접 지정하고 싶으시다면?

에이전트의 검색 과정을 생략하고, 특정 주소를 바로 사용하고 싶을 때의 방법입니다. 가장 확실하고 빠른 방법입니다.

1.  **설치 도우미 준비**:
    ```cmd
    winget install --id Git.Git -e --source winget
    ```
2.  **특정 주소로 즉시 설치**:
    ```cmd
    gemini skills install https://github.com/revfactory/harness --path skills/harness
    ```

---

## ✅ 설치 후 확인 및 사용

1.  **/skills list**: 내 컴퓨터에 설치된 도구 목록을 확인합니다.
2.  **/skills enable harness**: 도구를 활성화합니다.
3.  **지시하기**: "하네스를 사용해서 내 프로젝트의 협업 에이전트들을 설계해줘."

---

## 💡 Gemini 에이전트의 약속
"하네스"라는 이름의 다른 프로젝트가 있더라도 걱정하지 마세요. 저는 단순히 이름만 보는 것이 아니라, **그 도구가 제공하는 기능과 사용자의 목적이 얼마나 일치하는지**를 꼼꼼히 따져보고 결정합니다. 불확실할 때는 항상 사용자에게 확인을 구할 것입니다. 여러분은 결과만 기대해 주세요.
